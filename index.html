BUỔI 8

BIOS/UEFI — Hệ Thống Khởi Động

Hiểu về firmware đầu tiên chạy khi bật máy tính, cách vào BIOS/UEFI, và vai trò của nó trong bảo mật hệ thống

Giới thiệu BIOS vs UEFI POST Boot Order Bảo mật Thực hành
01

BIOS là gì?

🔧 Basic Input/Output System

BIOS (Basic Input/Output System) là chương trình firmware đầu tiên được thực thi khi bạn bấm nút nguồn máy tính. Nó nằm trên chip ROM của mainboard và có nhiệm vụ khởi động phần cứng, kiểm tra hệ thống, rồi tìm và khởi chạy hệ điều hành.

💡 Vị trí của BIOS

BIOS được lưu trữ trong chip ROM (Read-Only Memory) hoặc Flash Memory trên mainboard. Nó hoạt động độc lập với ổ cứng, vì vậy ngay cả khi không có HDD/SSD, BIOS vẫn chạy được.

⚙️ Vai trò của BIOS/UEFI
  • Khởi động phần cứng: Cấp nguồn và kích hoạt CPU, RAM, ổ cứng, GPU
  • POST (Power-On Self Test): Kiểm tra phần cứng có hoạt động bình thường không
  • Tìm hệ điều hành: Quét các thiết bị lưu trữ theo thứ tự Boot Order
  • Chuyển quyền điều khiển: Khi tìm thấy HĐH, BIOS giao quyền điều khiển cho bootloader của HĐH
02

BIOS cũ vs UEFI mới

Legacy BIOS
BIOS Cổ điển

Ưu điểm: Đơn giản, tương thích rộng với phần cứng cũ

Nhược điểm: Giao diện text, chậm, chỉ hỗ trợ ổ đĩa ≤2TB (MBR), không có Secure Boot

Modern UEFI
UEFI Hiện đại

Ưu điểm: Giao diện đồ họa, boot nhanh hơn, hỗ trợ ổ >2TB (GPT), Secure Boot, hỗ trợ chuột

Nhược điểm: Không tương thích với phần cứng quá cũ

📊 So sánh chi tiết
Tiêu chí Legacy BIOS UEFI
Giao diện Text only, keyboard GUI, hỗ trợ chuột
Tốc độ boot Chậm (20-30s) Nhanh (5-10s)
Phân vùng MBR (≤2TB, 4 primary) GPT (>2TB, 128 partition)
Bảo mật Không có Secure Boot Có Secure Boot, TPM
Yêu cầu Windows 11 Không hỗ trợ Bắt buộc
03

POST - Power-On Self Test

🔍 Quy trình kiểm tra phần cứng

POST là quá trình tự kiểm tra mà BIOS/UEFI thực hiện ngay sau khi bật máy. Nó kiểm tra từng linh kiện phần cứng xem có hoạt động bình thường không.

Bước 1: Kiểm tra CPU

Xác nhận CPU hoạt động và có thể thực thi lệnh

Bước 2: Kiểm tra RAM

Test các module RAM, đảm bảo không có lỗi memory

Bước 3: Kiểm tra Card đồ họa

Xác nhận GPU/iGPU hoạt động, hiển thị được hình ảnh

Bước 4: Kiểm tra các thiết bị I/O

Keyboard, mouse, ổ cứng, USB devices

Bước 5: Tìm bootable device

Quét các thiết bị theo Boot Order để tìm hệ điều hành

🔊 Tiếng Beep - Mã lỗi POST

Khi POST phát hiện lỗi, mainboard sẽ phát ra các tiếng beep khác nhau để thông báo. Đây là cách chẩn đoán lỗi phần cứng khi màn hình không hiển thị gì.

Mã Beep Ý nghĩa Cách khắc phục
1 tiếng ngắn ✅ POST thành công, hệ thống OK Không cần khắc phục
2 tiếng ngắn ⚠️ Lỗi RAM hoặc RAM không được cắm đúng Kiểm tra lại RAM, cắm lại hoặc thử thanh khác
3 tiếng ngắn ⚠️ Lỗi RAM (thường là bank đầu tiên) Thay RAM hoặc đổi slot
Liên tục dài 🚨 RAM không được phát hiện hoặc hỏng hoàn toàn Kiểm tra kỹ RAM, slot RAM, hoặc thay RAM mới
1 dài 2 ngắn 🚨 Lỗi card đồ họa (VGA) Kiểm tra kết nối GPU, thử cắm lại hoặc dùng iGPU
1 dài 3 ngắn 🚨 Lỗi card đồ họa hoặc không tìm thấy monitor Kiểm tra dây cáp màn hình, GPU
💡 Lưu ý

Mã beep có thể khác nhau tùy hãng mainboard (AMI, Award, Phoenix). Nên tra cứu manual của mainboard để biết chính xác.

04

Boot Order - Thứ tự khởi động

🎯 BIOS tìm hệ điều hành ở đâu?

Sau khi POST hoàn tất, BIOS sẽ quét các thiết bị lưu trữ theo thứ tự ưu tiên (Boot Order) để tìm bootloader của hệ điều hành. Thiết bị nào có bootloader hợp lệ đầu tiên sẽ được khởi động.

Boot Option #1
💾 SSD/HDD

Ổ cứng chính chứa Windows/Linux. Đây là lựa chọn mặc định khi boot bình thường.

Boot Option #2
🔌 USB Drive

USB boot để cài Windows, chạy Linux live, hoặc rescue tools. Ưu tiên cao khi cài hệ điều hành.

Boot Option #3
💿 CD/DVD

Đĩa quang (hiện nay ít dùng). Trước đây dùng để cài Windows từ đĩa.

Boot Option #4
🌐 Network (PXE)

Boot qua mạng LAN. Dùng trong môi trường doanh nghiệp để cài hệ điều hành từ xa.

💡 Khi nào cần thay đổi Boot Order?

Cài Windows/Linux: Đặt USB lên đầu để boot từ USB cài đặt

Sửa lỗi boot: Boot từ USB rescue để khắc phục bootloader hỏng

Dual boot: Chọn ổ đĩa nào boot trước

05

Bảo mật: Secure Boot & TPM 2.0

🔐 Secure Boot

Secure Boot là tính năng bảo mật của UEFI, chỉ cho phép khởi động hệ điều hành và bootloader được ký số (digitally signed) bởi nhà sản xuất tin cậy.

Mục đích: Ngăn chặn malware/rootkit chạy trước khi HĐH khởi động (boot-level attacks).

Lưu ý: Windows 11 yêu cầu BẮT BUỘC Secure Boot. Một số Linux distro cần tắt Secure Boot mới cài được.

🔑 TPM 2.0

TPM (Trusted Platform Module) là chip bảo mật phần cứng trên mainboard, lưu trữ khóa mã hóa và chứng chỉ bảo mật.

Chức năng:

  • Mã hóa ổ cứng (BitLocker)
  • Xác thực sinh trắc học (Windows Hello)
  • Ngăn chặn tampering (thay đổi phần cứng trái phép)

Windows 11: Yêu cầu TPM 2.0 bắt buộc.

⚠️ Nguy hiểm: Boot từ USB không rõ nguồn gốc

Nếu ai đó boot USB trên máy của bạn (khi không có mật khẩu BIOS), họ có thể:

  • Cài phần mềm gián điệp, keylogger
  • Đánh cắp dữ liệu (nếu ổ cứng không mã hóa)
  • Reset mật khẩu Windows (nếu không có BitLocker)

Giải pháp: Đặt mật khẩu BIOS + bật Secure Boot + mã hóa ổ cứng bằng BitLocker.

Thực hành

⌨️ Cách vào BIOS/UEFI

Cách 1: Nhấn phím khi khởi động (truyền thống)

Dell, Lenovo, HP
F2 hoặc F10

Nhấn liên tục ngay khi logo hiện ra

ASUS, MSI, Gigabyte
Del (Delete)

Desktop thường dùng Delete

Acer
F2 hoặc Del

Tùy model cụ thể

Boot Menu
F12 hoặc F11

Chọn thiết bị boot nhanh (không vào BIOS)

Cách 2: Vào UEFI từ Windows 10/11

Bước 1

Mở Settings (Win + I) → Update & Security → Recovery

Bước 2

Tìm mục "Advanced startup" → nhấn "Restart now"

Bước 3

Chọn Troubleshoot → Advanced options → UEFI Firmware Settings → Restart

⚠️ QUAN TRỌNG: CHỈ XEM, KHÔNG THAY ĐỔI!

Khi vào BIOS/UEFI lần đầu, CHỈ QUAN SÁT, không thay đổi bất kỳ cài đặt nào. Thay đổi sai có thể khiến máy không boot được hoặc hỏng phần cứng.

Nếu vô tình thay đổi gì, nhấn ESC hoặc chọn "Exit Without Saving" để thoát an toàn.

🗺️ Tour BIOS/UEFI - Các Tab chính
Tab MAIN (Thông tin chính)

📅 Ngày giờ hệ thống (System Date/Time)

💾 Danh sách ổ cứng được phát hiện (SATA, M.2)

⚙️ Thông tin BIOS version

Tab BOOT (Thứ tự khởi động)

🎯 Boot Priority / Boot Order: Thứ tự tìm HĐH

🔌 Boot Option #1, #2, #3,... (SSD, USB, DVD,...)

⚡ Fast Boot: Bật/tắt khởi động nhanh

Tab ADVANCED (Nâng cao - CHỈ XEM)

🧠 CPU Configuration: Tần số, số nhân, công nghệ ảo hóa (VT-x/AMD-V)

💾 Memory Configuration: Tần số RAM, XMP profile

⚠️ CẢNH BÁO: Không chỉnh overclock nếu không hiểu rõ!

Tab SECURITY (Bảo mật)

🔒 Set Supervisor Password: Mật khẩu BIOS (vào BIOS)

🔐 Secure Boot Control: Bật/tắt Secure Boot

🔑 TPM Configuration: Bật/tắt TPM 2.0

Tab EXIT (Thoát)

💾 Save Changes and Exit: Lưu thay đổi và thoát (F10)

🚫 Exit Without Saving: Thoát không lưu (ESC)

🔄 Load Setup Defaults: Khôi phục cài đặt mặc định

🔍 Kiểm tra phiên bản BIOS trên Windows

Không cần vào BIOS, bạn có thể kiểm tra phiên bản BIOS ngay trên Windows:

Cách 1: System Information

1. Nhấn Win + R → gõ msinfo32 → Enter

2. Xem dòng "BIOS Version/Date"

Cách 2: Command Prompt

1. Mở CMD (Win + R → cmd)

2. Gõ lệnh: wmic bios get smbiosbiosversion

💡 Khi nào cần cập nhật BIOS?

BIOS update rất NGUY HIỂM nếu làm sai (có thể brick mainboard). Chỉ nên update khi:

  • Mainboard không nhận CPU mới (cần update microcode)
  • Có lỗi nghiêm trọng mà BIOS update khắc phục
  • Cần bật tính năng mới (TPM, Resizable BAR,...)

Lưu ý: Không bao giờ tắt máy hoặc mất điện khi đang update BIOS!

07

Tóm tắt

📝 Những điều cần nhớ
  • BIOS/UEFI là chương trình firmware đầu tiên chạy khi bật máy, nằm trên chip mainboard
  • POST kiểm tra phần cứng, phát tiếng beep nếu có lỗi
  • Boot Order quyết định máy tìm HĐH ở đâu trước (SSD, USB, DVD, Network)
  • UEFI hiện đại hơn BIOS cũ: nhanh hơn, bảo mật hơn, hỗ trợ ổ >2TB
  • Secure Boot chỉ cho boot HĐH đáng tin cậy, ngăn malware
  • TPM 2.0 là chip bảo mật phần cứng, yêu cầu bắt buộc cho Windows 11
  • Mật khẩu BIOS ngăn người khác vào BIOS hoặc boot từ USB
⚠️ An toàn khi thực hành

Khi vào BIOS/UEFI lần đầu, CHỈ XEM, không thay đổi gì. Nếu thay đổi nhầm, nhấn ESC để thoát không lưu. Không bao giờ chỉnh Voltage, Frequency, hoặc Overclock nếu chưa hiểu rõ.
